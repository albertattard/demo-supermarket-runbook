# demo-supermarket-runbook

This repository is the reproducible source for creating the `demo-supermarket`
GitHub repository. It is not the supermarket application repository itself.

The generated target repository is created under `/tmp/demo-supermarket` by
running the Sociable Weaver runbook in `sw-runbook.yaml`.

## Repository Structure

- `sw-runbook.yaml` is the source of truth for the generated workflow.
- `run.sh` removes and recreates the local `/tmp/demo-supermarket` output by
  running `sw run --working-directory /tmp/demo-supermarket --verbose`.
- `fixtures/README.md` becomes the generated target repository README.
- `fixtures/LICENSE` becomes the generated target repository license.
- `fixtures/issues/` contains GitHub issue bodies used by the runbook.
- `fixtures/pr/` contains pull request bodies and generated PR URL outputs.
- `fixtures/repository-overlays/` contains staged file overlays copied into the
  target repository on feature branches.
- `README.md` is generated from `sw-runbook.yaml` and should not be edited
  manually.
- `/tmp/demo-supermarket`, when present, is generated output and should not be
  treated as source.

## Editing Guidance

Prefer editing `sw-runbook.yaml` and files under `fixtures/`.

Do not manually patch generated files in `/tmp/demo-supermarket` unless the task
is specifically to inspect or debug generated output. Instead, update the
runbook or fixtures and regenerate.

Do not manually edit root `README.md`; it is generated by Sociable Weaver and
manual edits will be overwritten.

When changing generated application content, place the intended final files in
the appropriate `fixtures/repository-overlays/<step>/` directory.

When changing issue or pull request text, edit the matching Markdown file under
`fixtures/issues/` or `fixtures/pr/`.

## Validation

After runbook or fixture changes, run:

```shell
./run.sh
```

This recreates the local `/tmp/demo-supermarket` output from scratch.

The runbook may use `gh` and create, delete, push, check, and merge GitHub
resources for the target `demo-supermarket` repository. Do not run the full
workflow against GitHub unless that external side effect is intended.

If only checking generated local files, inspect the relevant files under
`/tmp/demo-supermarket` after regeneration.

## Naming

Use `demo-supermarket-runbook` for this repository.

Use `demo-supermarket` only for the generated target repository.
